HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown
  • The header element appears at the top of the page and usually contains the navigation menu and introductory content
  • The paragraph element represents a block of text
  • The section element defines a section in the document and allows us to elegantly organize the content of a page
  • The image element contains information about images that are displayed on the webpage
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation

CSS

  • There are 3 ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • Inline CSS: applied to a specific HTML tag and styles one particular page element only
  • Internal style sheet: embedded directly into the head section of an HTML file. These CSS rules apply only to that page
  • External style sheet: the most common and preferred way. Allows developers to keep all their CSS rules in a separate file, making design changes easier
  • Separation of concerns: widely used term based on the idea that each section of code should have its own responsibility. Code is separated into different files based on use
  • A largin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • A period preceding a class name designates it as a class selector. Any HTML element assigned to that class will gain these properties

Git

  • git status: checks what branch we are currently on
  • git checkout main: switches to the branch called main
  • git pull origin main: pulls down the latest main branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds the changed files in our working directory to the staging area of Git
  • git commit -m "added code to HTML file": commits the changes and adds a short message about the changes we implemented
  • git pull origin main: it is good habit to always pull the base branch into your feature branch before pushing up any code -you should see a message in your terminal saying something like "already up to date"
  • git push origin feature/add-html: pushes up all of our changes that we made in the new branch
  • Next, navigate to the appropriate repository on GitHub. This is where we will merge our feature branch into the base branch
  • Once in the repository, navigate to the desired branchand click the "pull requests" tab at the top
  • Click the "new pull request" button. Give the PR a title and a short description, then click the "create pull request button"
  • Click the "merge pull request" button and confirm
  • Navigate back to the main branch in GitHub and you should see your file is now there
  • Next, go back to the terminal and enter the command git checkout main. Then enter the command git pull origin main. This updates our local reo with the new merged-in feature
  • Next we want to close our issue on GitHub since we have merged in our feature. Navigate to the "issues" tab, select that issue, and click the "close issue" button

JavaScript

  • Variables are containers that store values. You declare a variable with the "let" keyword
  • A semicolon at the end of a line indicates where a statement ends. It's only required when you need to separate statements on a single line
  • After declaring a variable, you can give it a value
  • You retrieve the variable by calling the variable name. You can change the assigned value of a variable later in the code
  • Variable data types: strings, numbers, boolean aka true/false, arrays, and object
  • Functions are a way of packaging functionality that you wish to reuse
  • Conditionals are code structures used to test if an expression returns true or false
  • An operator is a mathematical symbol that produces a result based on two values or variables
  • The return statement tells the browser to return the result
  • Events are code structures that listen for an activity in the browser and run code in response. An example is the click event
